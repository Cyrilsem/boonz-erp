BUG-1 · B-04 Qty Adjust to Zero — Not Saving MODULE B: PACKING · /field/packing/[id]
While editing pickup qty, its not saving 0 as pick up qty
What she tested: Set the pick quantity to 0 in the packing detail screen and tapped "Packed" — the value was not persisting as 0.

Likely root cause: The packing form may treat qty=0 as falsy/empty and either revert to the original quantity or block the save. This is a validation or mutation issue in the packing confirmation logic. Check if the refill_dispatching row update rejects zero-quantity writes.
To investigate: Check the packing detail mutation handler — the code likely has a guard like if (!qty) return that filters out 0 as falsy. Change to if (qty === undefined || qty === null) return. Also check if Supabase RLS or a DB constraint rejects qty = 0 on refill_dispatching.
BUG-2 · F-01 Zigi Sea Salted Not Appearing — Default Inventory View MODULE F: INVENTORY · /field/inventory
Zigi is not showing
What she tested: Default inventory view (Active filter, <=7 days expiry) — Zigi Sea Salted (seeded with expiry 18 Apr 2026 = today) is not visible.

Likely root cause: The item expires today (Apr 18). The query likely uses expiration_date < CURRENT_DATE for "expired" (strictly less than = yesterday or earlier), meaning today's items fall in a grey zone — not yet in "expired" and also not in "active near-expiry." The expiry_warning column on the seeded row is NULL (not 'expired'), so it does not match the Expired filter either.
To investigate: Run SELECT id, product_name, expiration_date, expiry_warning, qty_in_stock FROM warehouse_inventory WHERE product_name ILIKE '%zigi%'; to confirm the row exists and check its expiry_warning value. The inventory query likely needs expiration_date <= CURRENT_DATE for the expired filter (inclusive of today). Also verify the item wasn't consumed by the dispatching workflow.
BUG-3 · F-02 Expired Filter — Not Showing Items MODULE F: INVENTORY · /field/inventory
Its not showing
What she tested: Tapped the "Expired" filter pill in inventory — no items appeared (expected: Mezzmix Chocolate + Zigi Sea Salted).

Likely root cause: Same boundary issue as BUG-2. Additionally, Mezzmix Chocolate (seeded expiry 9 Apr) should definitely be showing — if neither item appears under "Expired," the filter query may have a deeper issue, possibly the expiry_warning = 'expired' column filter is being used instead of date comparison, and neither seeded row has expiry_warning = 'expired' set (both were seeded with NULL).
To investigate: Run SELECT product_name, expiration_date, expiry_warning FROM warehouse_inventory WHERE expiration_date <= CURRENT_DATE; — if rows exist here but the Expired filter shows nothing, the UI query is using expiry_warning = 'expired' instead of date comparison. Either update the seed data to set expiry_warning = 'expired' on those rows, or fix the filter query to compare against expiration_date directly.
