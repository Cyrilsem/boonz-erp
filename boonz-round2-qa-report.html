Bugs & Design Gaps Found in Round 2

BUG-4 · Critical
Partial receive overwrites ordered_qty — shortfall permanently lost

When receiving fewer units than ordered (e.g. 3 of 6), the receive flow updates ordered_qty to the received quantity. There is no received_qty column. The original order quantity is permanently lost. The PO moves to "Received" with no Partial status or shortfall record.

BUG-5 · Major
Removal reason (Expired / Damaged / Other) not persisted to DB

The Removals page collects a reason per line (UI dropdown: Expired, Damaged, Other) but the handleSubmit function never includes the reason in the pod_inventory insert. The reason is silently dropped, making post-hoc auditing impossible.

GAP · Not Implemented
F-04: Low-stock badge (≤5 units) not implemented

No visual indicator exists for items with 5 or fewer units in the warehouse inventory page. Only a "Qty Low" sort option exists — no badge or alert styling.

GAP · Not Implemented
F-05: CSV export not implemented on inventory page

No export button, CSV download, or data export mechanism exists anywhere on /field/inventory.

GAP · Design
No machine-to-machine transfer flow

There is no route or UI for transferring products between machines. The closest flow is the removals page which only logs removal from a single machine — no destination machine can be specified.

GAP · Design
No physical warehouse return flow

Removals page sets pod_inventory.current_stock = 0, status = "Removed / Expired". It does not create a warehouse_inventory entry for returned goods. Physical product returned to warehouse is not reflected in warehouse stock.

GAP · Design
No Partial PO status — partial receives appear fully received

A PO received 10/10 Hazelnut + 3/6 Aquafina shows as "Received" with no indication of the 3-unit shortfall. Orders list shows "No pending orders" after partial receive. Buyers have no visibility into unfulfilled quantities.

GAP · Design
C-06: No re-dispatch mechanism for incomplete packing

If warehouse only packs 3 of 5 dispatch lines, the machine never appears in the driver pickup list (requires ALL lines packed). There is no "force pickup with partial pack" or "re-queue unpacked lines for next dispatch" workflow.

GAP · Design
A-03: No in-app dark mode toggle

Dark mode is CSS-only via OS/browser preference. No in-app toggle exists. Users cannot switch mode without changing OS settings.
