01The metaphor

BEFORE

A working spare key under the doormat

Anyone with a session token (authenticated) could walk in through the back door and call the legacy rename function. SECURITY DEFINER meant it ran as the database owner — full bypass of RLS — even if the canonical replacement repurpose_machine already existed.

  • The function was still capable of writing to machines with no role check.
  • If a forgotten FE bookmark or a third-party script still hit this RPC, the rename would succeed silently.
  • Two write paths to machines existed for the same business operation — a violation of Article 1.
AFTER

Spare key removed; door welded for FE; manager-only override

The function is now SECURITY INVOKER, and EXECUTE has been revoked from both anon and authenticated. The FE cannot call it. service_role keeps EXECUTE for the 90-day monitor as an emergency escape — visible in the migration log if anyone uses it.

  • One write path to machines for identity transitions: repurpose_machine.
  • Function comment now reads DEPRECATED 2026-04-25 with the scheduled DROP date.
  • Calendar reminder for 2026-07-24: revoke service_role EXECUTE, then DROP.
